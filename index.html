Diferencia entre Internet y la World Wide Web

El internet es la red global de computadoras interconectadas que permite el intercambio de datos y comunicacion a traves de diversos protocolos.
La World Wide Web es un sistema de información que utiliza Internet para acceder y compartir documentos y recursos a través de navegadores web, utilizando el protocolo HTTP.
Podriamos decir que la diferencia entre estos conceptos es parecida a la diferencia entre una carretera y los autos que la recorren. Siendo internet la carretera y WWW el conjunto de páginas y servicios que viajan sobre esa carretera

Cuales son las partes de un URL?

  1. Esquema: Indica el protocolo (ej. http, https).
  2. Dominio: Nombre del sitio o dirección IP (ej. www.example.com).
  3. Puerto (opcional): Especifica el puerto del servidor (ej. :80).
  4. Ruta: La dirección del recurso en el servidor (ej. /path/to/page).
  5. Consulta (opcional): Parámetros enviados al servidor (ej. ?id=123).
  6. Fragmento (opcional): Navegación dentro de la página (ej. #section1).

Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  1. GET: Solicita un recurso sin modificarlo.
  2. HEAD: Similar a GET, pero solo obtiene los encabezados de la respuesta.
  3. POST: Envía datos al servidor para crear un recurso.
  4. PUT: Envía datos al servidor para actualizar o crear un recurso.
  5. PATCH: Modifica parcialmente un recurso existente.
  6. DELETE: Elimina un recurso en el servidor.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

POST: Se usa porque envía los datos de manera segura y permite enviar datos de gran tamaño, además de no mostrar los datos en la URL.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

GET: Se utiliza para solicitar y obtener el contenido de la página.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Significa que la solicitud fue exitosa y el servidor devolvió el recurso solicitado. No ocurrió ningún error.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Depende, el error 404 indica que el recurso solicitado no fue encontrado. Si el recurso debería existir, es responsabilidad del desarrollador corregirlo. Si no, puede ser que el usuario haya introducido una URL incorrecta

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, el error 500 indica un problema interno del servidor, lo que sugiere un problema en el código o en la configuración del servidor que el desarrollador debe solucionar
